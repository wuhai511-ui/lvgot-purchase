## 一、项目概述

### 1.1 背景

旅购通运营后台原定位为对接钱账通（QZT）的支付网关系统。本次重构将其重新定位为**商户工作台的后台支撑和服务系统**，专注于商户管理、导游审核、功能权限配置等运营管理职能，不再直接对接钱账通。

### 1.2 重构目标

1. **数据架构**：采用共享数据库 + tenant_id 隔离方案，支持运营后台管理全部商户数据
2. **权限模型**：简化为超级管理员和普通管理员两级
3. **功能范围**：保留商户管理、导游管理、门店管理、旅行社管理、交易数据查看、分账配置
4. **实施方式**：直接在现有 Vue 代码基础上重构

## 三、权限模型

采用两级管理员角色：

| 角色 | 说明 | 权限范围 |
|------|------|---------|
| **超级管理员** | 系统最高权限 | 无限制，可操作所有模块 |
| **普通管理员** | 运营人员 | 查看/操作除财务核心外的所有模块（充值、提现记录查看） |

## 四、功能模块设计

### 4.1 租户管理（商户管理）

**功能说明**：管理商户（租户）的基本信息、功能权限配置

**现有表结构**：使用 `merchants` 表

**核心字段**：
- `id`、`out_request_no`、`register_name`（商户名称）
- `legal_mobile`、`legal_name`、`legal_id_card`
- `license_no`、`enterprise_type`（企业类型：1-个体工商户、2-企业、3-个人）
- `split_role`（分账角色：merchant/store/agency/guide/driver/other）
- `status`（商户状态：PENDING/APPROVED/REJECTED/ACTIVE）
- `created_at`、`updated_at`

**运营后台操作**：
- 商户列表查询（支持按名称、手机、状态筛选）
- 商户详情查看
- 商户状态变更（启用/禁用）
- 功能权限配置（预留字段）

### 4.2 导游管理

**功能说明**：导游开户申请、导游证审核（直接通过/驳回）

**核心逻辑**：
- 导游属于 `split_role = 'guide'` 的商户
- 导游证字段：`guide_cert_no`（导游证号）、`guide_cert_img`（导游证图片）
- 审核流程：运营人员直接审核通过/驳回，无需外部系统

**运营后台操作**：
- 导游申请列表（筛选 `split_role = 'guide'` 且 `status = 'PENDING'`）
- 导游详情查看（基本信息 + 导游证照片）
- 导游证审核（通过/驳回，驳回需填写原因）
- 已审核导游列表

### 4.3 门店管理

**功能说明**：门店信息管理

**现有表结构**：使用 `stores` 表

**核心字段**：
- `id`、`store_id`（门店编号，格式 LVGSyyyyMMddnnnnnn）
- `store_name`、`account_no`
- `merchant_id`（所属商户）
- `status`（ACTIVE/DISABLED/DELETED）

**运营后台操作**：
- 门店列表查询（支持按商户、门店名称筛选）
- 门店详情查看
- 门店终端绑定（`store_terminals` 表）

### 4.4 旅行社管理

**功能说明**：旅行社/导游团队管理

**现有表结构**：使用 `tour_groups` 表

**核心字段**：
- `id`、`tour_no`、`tour_name`、`route_name`
- `merchant_id`（所属商户）
- `start_date`、`end_date`、`people_count`
- `guide_id`、`driver_id`（导游/司机关联商户ID）
- `status`（ACTIVE/DISABLED/DELETED）

**运营后台操作**：
- 旅行社/团队列表查询
- 团队详情查看
- 团队成员管理（导游、司机）

### 4.5 功能权限管理

**功能说明**：租户功能权限配置

**设计思路**：
- 在 `merchants` 表扩展 `features` JSON 字段，或新建 `merchant_features` 表
- 功能权限包括：是否开通分账、是否开通提现、是否开通对账等

**权限项设计**：
```json
{
  "enable_split": true,
  "enable_withdraw": true,
  "enable_reconciliation": true,
  "enable_store_management": true,
  "max_stores": 10
}
```

### 4.6 财务报表

**功能说明**：账户流水、充值/提现记录、对账

**现有表结构**：
- `transactions` — 交易流水（充值、提现）
- `accounts` — 账户余额
- `reconciliation_tasks` — 对账任务

**运营后台操作**：
- 账户流水查询（支持按商户、时间、交易类型筛选）
- 充值记录查询
- 提现记录查询
- 对账任务管理

### 4.7 分账管理

**功能说明**：分账规则和分账模板配置

**现有表结构**：
- `split_rules` — 分账规则
- `split_rule_items` — 分账规则明细
- `split_templates` — 分账模板

**运营后台操作**：
- 分账规则列表/新增/编辑/删除
- 分账模板列表/新增/编辑/删除

## 五、数据隔离设计

### 5.1 核心原则

1. **所有业务表增加 `tenant_id` 字段**（系统配置表除外）
2. **商户端 API**：强制注入当前登录商户的 `tenant_id`，仅可操作自身数据
3. **运营后台 API**：不注入 `tenant_id`，可按需筛选全部或指定商户数据

### 5.2 表结构调整

需要增加 `tenant_id` 字段的表：

| 表名 | 说明 | tenant_id 含义 |
|------|------|---------------|
| `merchants` | 商户表 | 自身（自关联） |
| `stores` | 门店表 | 所属商户 |
| `tour_groups` | 旅行社表 | 所属商户 |
| `split_rules` | 分账规则表 | 所属商户 |
| `split_templates` | 分账模板表 | 创建商户 |
| `transactions` | 交易流水表 | 所属商户 |
| `accounts` | 账户表 | 所属商户 |

### 5.3 API 分层

```
商户工作台 API（前端）      运营后台 API（前端）
       │                          │
       ▼                          ▼
  tenant_id 强制注入           tenant_id 可选筛选
       │                          │
       └──────────┬───────────────┘
                  ▼
            共享数据库
```

## 七、实施计划

### 阶段一：数据层改造
1. 数据库表增加 `tenant_id` 字段
2. 编写数据库迁移脚本

### 阶段二：API 层改造
1. 商户端 API 强制注入 tenant_id
2. 运营后台 API 支持可选筛选
3. 新增 admin 专属 API

### 阶段三：前端重构
1. 路由调整（运营后台独立路由）
2. 页面组件重构（按新功能模块）
3. API 调用适配

## 九、验收标准

1. 运营后台可查看所有商户数据
2. 商户工作台仅可查看自身数据
3. 导游证审核功能完整（申请列表、审核通过/驳回）
4. 功能权限配置生效
5. 门店、旅行社、分账规则、分账模板管理功能正常
6. 财务报表（流水、充值、提现、对账）功能正常
