# 旅购通商户工作台 — 设计规格

> 生成时间：2026-03-30
> 状态：已确认

---

## 1. 项目概述

**项目名称：** 旅购通商户工作台
**项目类型：** Vue3 + Vite + Element Plus 的 PC Web 管理后台
**核心目标：** 旅购通商户通过 PC 端工作台，调用钱账通接口，完成开户、绑卡、分账、提现、对账等资金操作
**目标用户：** 旅购通系统商户（导游、司机、旅行社等）

---

## 2. 账户体系

```
商户（手机号登录）
  └─ 钱账通主账户（一个商户一个）
       ├─ 拉卡拉支付账户（余额）
       └─ 银行内部户（余额）
```

- 账户概览显示：主账户余额 = 拉卡拉余额 + 银行内部户余额
- 多账户切换：暂不支持多主体，钱账通账户为主

---

## 3. 功能模块

### 3.1 登录

- **方式：** 手机号 + 验证码（Mock，任意验证码均可登录成功）
- **登录后判断：**
  - 未开户 → 引导进入开户申请流程
  - 已开户 → 进入账户概览
- **Token：** JWT，存储在 localStorage

### 3.2 账户与资金

| 页面 | 功能 |
|------|------|
| 账户总览 | 展示主账户余额 = 拉卡拉余额 + 银行内部户余额 |
| 开户申请 | 填写商户信息表单 → 上传证件照片（身份证正反面、营业执照）→ 调用 file.upload.commn 接口获得 file_key → 提交到 BFF → BFF 保存商户信息（状态=待审核）→ BFF 调用钱账通 `/api/merchant/apply` 获取 H5 开户 URL → 前端 window.location.href 跳转 → 用户在钱账通 H5 做认证 → 钱账通回调 `/api/merchant/callback` 通知结果 → BFF 主动查询钱账通确认账户信息 → 更新商户状态 |
| 银行卡管理 | 绑卡：商户填写卡号 → BFF 调用钱账通 `/bank/bind` → 返回结果；解绑；查看已绑卡列表；未绑卡则提现按钮禁用 |
| 充值 | 账户充值转账（手动） |
| 提现 | 需先绑卡；展示余额和手续费说明（来自钱账通配置）；支持自动提现规则配置（金额门槛/周期） |

### 3.3 交易管理

| 页面 | 功能 |
|------|------|
| 付款订单 | 手动分账：选择付款账户 → 选择收款方 → 填写金额 → 提交 → BFF 调用钱账通 `/trade/pay` → 返回结果 |
| 分账规则 | 配置自动分账规则（按比例/固定金额），交易订单自动触发分账 |
| 分账记录 | 查看所有分账历史（时间、金额、状态、收款方） |
| 交易消息明细 | 钱账通实时推送交易通知到 BFF → BFF 存库 → 前端分页查询，支持按商户号、交易流水号筛选 |

### 3.4 门店管理

- 门店列表：绑定门店编号、终端编号、关联账户
- 绑定/解绑门店（解绑需二次确认）

### 3.5 系统设置

| 页面 | 功能 |
|------|------|
| 员工管理 | 新增员工（手机号、姓名、部门）；编辑；禁用；员工登录后可代表商户操作 |
| 部门管理 | 增删改部门 |
| 角色管理 | 创建角色（如：财务/操作员/管理员），每个角色绑定可访问的菜单 |
| 权限管理 | 给角色分配具体操作权限 |

## 4. 前后端接口（前端 ↔ BFF ↔ 钱账通）

| 流程 | BFF 接口 |
|------|---------|
| 登录 | `POST /api/merchant/login` |
| 开户申请 | `POST /api/merchant/apply` → 返回钱账通 H5 URL |
| 开户回调 | `POST /api/merchant/callback`（钱账通→BFF） |
| 文件上传 | `POST /api/file/upload` → 返回 file_key |
| 绑卡 | `POST /api/bank/bind` |
| 银行卡列表 | `GET /api/bank/list` |
| 付款/分账 | `POST /api/trade/pay` |
| 分账规则 CRUD | `GET/POST/PUT/DELETE /api/split/rules` |
| 分账记录 | `GET /api/split/records` |
| 提现 | `POST /api/account/withdraw` |
| 自动提现规则 | `GET/POST/PUT /api/account/auto-withdraw` |
| 账户信息 | `GET /api/account/info` |
| 充值 | `POST /api/account/recharge` |
| 交易消息列表 | `GET /api/trade/notifications` |
| 员工 CRUD | `GET/POST/PUT /api/employee` |
| 部门 CRUD | `GET/POST/PUT /api/department` |
| 角色 CRUD | `GET/POST/PUT /api/role` |
| 权限 CRUD | `GET/POST/PUT /api/permission` |

## 9. 设计决策记录

| 决策 | 选项 | 理由 |
|------|------|------|
| 登录验证方式 | Mock 验证码 | 暂不接入短信平台，商户直接登录 |
| 账户体系 | 一商户一钱账通主账户 | 当前业务规模不需要多主体 |
| 开户流程 | 跳转钱账通 H5 做认证 | 符合钱账通合规要求（人脸识别等） |
| 开户数据同步 | 回调 + 主动查询双保险 | 保证数据一致性 |
| 绑卡流程 | 商户填卡号 → BFF 调用钱账通接口 | 无需跳转，直接返回结果 |
| 分账 | 支持手动 + 自动规则两种模式 | 满足不同商户需求 |
| 交易消息 | BFF 接收推送存库，前端分页查询 | 便于商户追溯历史数据 |
| 员工权限 | RBAC 模型（角色 + 权限） | 员工可代表商户操作，操作记录可追溯 |
